# App 营销绘图 Prompt 生成 & 生图

将 App 营销信息转化为高质量中文绘图 prompt，并完成生图。最终产出图片文件。

## 索引状态

> **当前风格索引：2**（到18归1，仅用风格矩阵兜底时更新）
> **当前构图索引：9**（到11归1，每次生成后始终更新）
> **当前主体物索引：1**（到3归1，每次生成后始终更新）

**三索引的位置：** 风格索引在 references/style-matrix.md 顶部，构图索引在 references/composition-matrix.md 顶部，主体物索引在本文件顶部。更新时修改对应位置。

手动覆盖语法：用户附加 `风格=CG厚涂 色调=暖棕色调 构图=宏大仰视式 光线=丁达尔效应 主体=具体描述 品牌层级=高端` 等指令时，跳过对应维度的推导，直接使用指定值。

## 输入信息与 Logo 处理

按优先级提取：应用名称 → 一句话描述 → 应用简介 → 应用类型 → 宣传素材 → 参考图（可选）→ 必须包含元素（可选）→ Logo 图片（可选）

当用户提供 Logo 图片时，从中提取「Logo 视觉指纹」，输出格式：
```
Logo 视觉指纹：
- 主色调：[颜色1]、[颜色2]
- 设计风格：[扁平/3D/线描/...]
- 核心元素：[图标+文字/纯图标/...]
- 品牌气质：[活泼/专业/可爱/...]
```
Logo 提取的主色调和设计风格直接用于 prompt，优先级最高。与分类规则冲突时以 Logo 为准。

## 输出格式

默认产出应用商店封面图 prompt，用户明确要求其他场景时才生成对应版本。

## 风格矩阵注入规则

所有 prompt 以 `16:9横图` 开头。生成前按以下7步执行：

1. **检查手动覆盖（最高优先级）** — 用户附加了风格/色调/构图/光线指令，直接使用
2. **分类规则优先** — 用户提供了应用分类且在 category-visual.md 中有对应视觉指纹，则使用分类规则，跳过风格矩阵
3. **风格矩阵兜底** — 分类未收录或未提供分类时，读取 references/style-matrix.md，按当前索引取风格和色调
4. **构图索引始终生效** — 读取 references/composition-matrix.md，按当前索引取构图和光线
5. **注入维度值** — 视觉风格和色调优先取分类规则，兜底取风格矩阵；构图和光线始终取构图矩阵
6. **主体物选取** — 当前主体物索引 mod 3 选取变体（1→A, 2→B, 0→C）
7. **更新索引** — 构图索引和主体物索引始终+1；风格索引仅在使用矩阵兜底时+1

## Prompt 构建规范

### 语言约束

- 全部中文，不得出现英文关键词或括号权重语法
- 不输出负面提示词
- 不得出现具体应用 Logo、品牌标志、商标图案

### 内容安全约束（硬性）

**人物禁令：**
- 默认不出现任何人物，画面主体为物件、符号或场景
- 完全禁止：真人、现实人物肖像、摄影级写实人物、抽象几何剪影、概念艺术插画人物
- 仅当叙事无法用物件/符号表达（如社交、直播类 App 的核心体验就是人与人的连接）时才允许出现人物

**法律合规：**
- 禁止具体国家货币符号、币种文字、纸币硬币样式；涉及"货币"主体时必须为中性通用表达
- 禁止知名 IP 形象、版权角色、卡通人物（如米老鼠、皮卡丘等）
- 禁止误导性医疗表述、金融收益承诺、赌博暗示
- 禁止暴力、血腥、色情、歧视性内容
- 禁止 LGBT 相关视觉元素、彩虹旗等符号、女扮男装、男扮女装

**视觉成熟度（硬性）：**
- 除母婴、幼儿类产品外，画面整体气质必须成熟，不得出现低龄化视觉特征
- 低龄化标志（非母婴/幼儿产品禁止）：Q版造型、卡通风格、泡泡字体感、幼儿园配色（高饱和彩虹色乱炖）、幼稚化图形语言

**画面纯净：**
- 禁止中文、英文、数字、Logo、水印、标题文字
- 禁止手机、平板、电脑、电视等电子设备外观（工具类 App 可用屏幕内容，但不能出现完整设备外形）
- 禁止 AI 味的图片特征：过度光滑塑料感皮肤、诡异打光、千篇一律的AI艺术脸、过度对称构图、不自然的完美感
- 禁止画中画、分屏、多画幅拼贴、内嵌相框

### 结构模板

`[主体+动作]，[构图/取景]，[场景/环境]，[风格]，[光线]，[技术参数]`

光线模块对生图影响极大，每个 prompt 必须包含具体光线描述。

### 正向质量词

每个 prompt 必须注入（从根源上规避低质量画面）：

**画质保证词（必选）：** 高清，画面清晰锐利，构图完整，主体完整呈现，精准对焦

**质感提升词（按场景选择）：**
- 防止低龄幼稚：成熟视觉风格、专业插画质感
- 防止廉价感：精致质感、高端视觉、细节丰富
- 防止负面氛围：正向情绪、活力氛围、积极场景

**负面约束词（应用商店封面图必选）：** 禁止出现任何文字、禁止出现标题、禁止出现副标题、禁止出现宣传语

### 参考图与元素约束

有参考图时，prompt 末尾追加：`参考附图的整体风格、色调与构图氛围`
有必须包含元素时追加：`画面中必须包含：[用户输入的元素描述]`

### 画面质量三原则

1. **防止画面零碎** — 前景、中景、背景各只一个主要元素，必须包含留白区域
2. **场景构图策略** — 应用商店封面图：宏大场景感，主体占比多档可选（15-20%/45-60%/75-90%）
3. **突出主体** — 主体有明确打光（边缘光/逆光/聚光），背景虚化，形成色彩或明暗对比

人物禁现原则：默认画面只有物件/符号/场景，不出现任何人物。仅社交/直播类确属核心体验无法用物件表达时可例外，使用抽象剪影（≤3色块无五官）

## 品牌层级识别

在分类视觉指纹之上叠加品牌层级修正（优先级仅次于 Logo 提取）：

| 层级 | 识别信号 | 修正方向 |
|------|---------|---------|
| 高端 | 腾讯/网易/字节/米哈游等大厂；原版IP/正版授权；十年经典/国民级 | 注入：电影级质感、精致光影、高端视觉、克制配色。禁用：卡通、Q版、糖果色、可爱、萌、幼稚 |
| 精品 | RAZ/BBC/迪士尼/DK等国际授权；海外引进；专业机构背书 | 注入：精品插画感、国际风格、成熟视觉风格。禁用：卡通、Q版、可爱、萌、幼稚、糖果色 |
| 标准 | 无以上信号 | 不做额外修正 |

识别成功时在 prompt 中注入对应修正词并强制排除禁用词。用户可手动覆盖：`品牌层级=高端/精品/标准`。

层级与分类的优先级：Logo提取 → 品牌层级修正 → 三级分类微调 → 一级分类基础 → 风格速查表（兜底）

## 信息补全机制

核心原则：不追问用户，全部自主推断，推断过程透明化。

### Step 1：信息分级

| 等级 | 判断标准 | 处理策略 |
|------|---------|---------|
| 充足 | 名称+分类+简介 | 直接生成，标注推断项 |
| 一般 | 名称+分类 | 名称语义解析+分类惯例，补全后生成 |
| 稀少 | 只有名称或只有分类 | 双引擎推断，补全后生成 |
| 极少 | 一句模糊描述 | 生成3个差异化风格方向供选择 |

### Step 2：名称语义解析

| 名称关键词 | 推断方向 | 视觉倾向 |
|---------|---------|---------|
| 龙/兽/精灵/王国/魔法/神/战 | 奇幻冒险游戏 | 高饱和暖色、史诗插画、戏剧性构图 |
| 学/教/宝/启/课/题/考 | 教育/儿童 | 柔黄嫩绿、卡通插画、亲切温暖 |
| 钱/财/金/贷/理/股/账 | 金融商务 | 深蓝金色、简洁专业、信任感 |
| 聊/友/圈/社/群/话/见 | 社交 | 珊瑚橙暖粉、人物互动、温暖活跃 |
| 音/乐/歌/听/播/声/曲 | 音乐娱乐 | 律动紫蓝、艺术感、动感视觉 |
| 英文/字母缩写/数字组合 | 科技工具 | 冷色系、极简UI感、科技感 |

名称无法匹配时，根据整体语感推断（硬朗→游戏/工具；柔软→生活/社交；严肃→商务/金融）。

### Step 3：极简兜底

信息极少时，生成3个有明显差异的风格方向（色调/风格/氛围各不相同），供用户选择。

### Step 4：推断透明化

每次生成前输出「信息解析卡」：
```
信息解析卡：
- 已知：[用户提供的信息]
- 推断：[每项推断内容及依据]
  · 游戏类型 → 开放世界RPG（依据：名称含"王国"）
  · 品牌主色 → 深紫+金色（依据：游戏-开放世界分类推荐色）
  · 视觉风格 → 史诗概念艺术（依据：一级分类"游戏"渲染风格）
  · 品牌层级 → 标准层级（依据：无触发词）
```

优先级：Logo提取 > 用户明确说明 > 三级分类推荐色 > 名称语义推断 > 一级分类基础

### Step 5：主体物语义推导

从应用信息按以下四个问题推导主体物：

**Q0. 差异化过滤** — 这个 App 和同类相比最独特的功能/体验是什么？用"独有的"约束 Q2 的物件选择。

行业通用符号可用但需优化精致度（场景层次、光影、材质），避免 icon 感：

| 分类 | 通用符号 | 优化方向 |
|------|---------|---------|
| 房产 | 钥匙、房子轮廓、握手 | 加入场景/质感/光影提升精致度 |
| 教育 | 书本、铅笔、灯泡 | 加入人物/情感/叙事 |
| 金融 | 金币、钱袋、上涨箭头 | 加入场景/情绪/叙事感 |
| 音乐 | 耳机、音符、麦克风 | 加入人物/沉浸感 |
| 社交 | 对话气泡、握手、人群 | 加入人物/真实社交场景 |

**Q1. 核心动作** — 用户使用时在做什么？
**Q2. 代表物件** — 最具代表性的视觉元素（必须通过 Q0 过滤）
**Q3. 视觉化方向** — 如何表达最能传达价值和情绪？
**Q4. 叙事三要素** — 把功能转化为可读故事：
- 人物 → 目标用户具体形象（年龄/状态/处境）
- 处境 → 使用 App 前后的关键节点
- 转变 → 画面定格的那一秒（问题被解决的瞬间或目标达成的状态）

叙事三要素让看图的人 3 秒内能读懂"谁 + 在做什么 + 结果是什么"。

**视觉化变体生成**（每个变体必须包含至少3层场景、2个动作、3层光影、2个材质、2个氛围词）：

| 变体 | 叙事节点 | 画面定格 |
|------|---------|---------|
| A | 痛点时刻 | 使用 App 前，问题最尖锐的那一秒 |
| B | 转变时刻 | 使用 App 中，问题被解决的瞬间 |
| C | 结果时刻 | 使用 App 后，目标达成的情感状态 |

输出格式：
```
· 差异化功能 → [Q0推导结果]
· 核心动作 → [Q1推导结果]
· 代表物件 → [Q2推导结果]（主体物）
· 叙事三要素 →
  · 人物：[目标用户具体形象]
  · 处境：[关键节点描述]
  · 转变：[画面定格的那一秒]
· 视觉化变体 →
  · 变体A：[痛点时刻]
  · 变体B：[转变时刻]
  · 变体C：[结果时刻]
· 人物角色 → [保留/省略] + 理由
· 本次使用 → 变体[X]（由主体物索引 mod 3 决定）
```

## 二次优化（质量自检前执行）

生成 prompt 后、进入自检前，按以下维度逐项检查优化，仅修改命中问题的维度：

| # | 维度 | 判断逻辑 | 优化动作 |
|---|------|---------|---------|
| 1 | 主体具象化 | 描述是否过于抽象 | 替换为具体可画视觉主体 |
| 2 | 分类特征强化 | 分类关键词是否出现；禁用词是否混入 | 注入分类特征词；移除禁用词 |
| 3 | 光线与构图协同 | 光线是否与构图类型匹配 | 替换为构图矩阵对应的光线 |
| 4 | 色调落地 | 色调是否泛指（"明亮色调"） | 替换为分类推荐主色 |
| 5 | 堆叠密度 | 是否少于5个分号段 | 补充缺失层次 |
| 6 | Logo和App图标禁用 | 是否出现图标/界面描述 | 替换为场景/氛围描述 |
| 7 | 构图差异化 | 同批次是否重复 | 为不同应用匹配差异化方案 |

优化后输出摘要：
```
【二次优化摘要】
- [维度名]：「原描述」→「优化后描述」
- [维度名]：无需优化
```

**批量差异化规则：** 多个应用同时生成时，先列视觉占用表，检查构图和人物形式是否重复，冲突则换方案。

## 质量自检流程

每个 prompt 生成后立即执行10维度自检（每项 0-5 分）：

| # | 维度 | 0分示例 | 5分示例 |
|---|------|--------|--------|
| 1 | 主体清晰度 | "展示功能价值" | "发光精灵蛋悬浮于魔法阵中央" |
| 2 | 光线描述 | 无光线词 | "戏剧性魔法逆光，金色粒子光晕环绕" |
| 3 | 构图完整性 | 无构图描述 | "仰视广角，主体居三分法交叉点" |
| 4 | 风格一致性 | 风格词与分类矛盾 | 风格词与分类完全吻合 |
| 5 | 构图场景感 | 图标用宏大感/海报用极简居中 | 构图策略与类型匹配 |
| 6 | 品牌色落地 | "彩色""好看的颜色" | "深紫星空蓝为主，金色边缘光点缀" |
| 7 | 可生成性 | 抽象情感词无视觉对应 | 每个词都有视觉画面 |
| 8 | Logo禁用检查 | 出现Logo/App图标/界面 | 仅有场景/氛围/用户行为 |
| 9 | 差异化识别度 | 通用符号无精致度 | 一眼识别核心功能，该 App 独有 |
| 10 | 叙事完整性 | 纯概念画面无人物无处境 | 人物+处境+转变三要素清晰 |

评分细则（第9项）：0分=纯icon无场景；3分=能识别行业但无法区分App；5分=能识别具体功能且是差异化卖点
评分细则（第10项）：0分=纯概念；3分=有人物但叙事不清晰；5分=3秒可读懂故事

总分低于 35 分（满分 50）自动修正。得分 ≤2 的维度按以下修正：

| 不合格维度 | 修正动作 |
|---------|---------|
| 主体清晰度不足 | 替换为具体可视化场景 |
| 缺少光线描述 | 应用商店封面图→戏剧性逆光 |
| 构图不完整 | 应用商店封面图→对角线三分法 |
| 风格不一致 | 替换为分类渲染风格关键词；高端/精品层级移除禁用词 |
| 场景感不足 | 追加「电影级广角，渺小主体衬宏伟背景」 |
| 品牌色缺失 | 从 category-visual.md 取分类推荐主色 |
| 可生成性差 | 替换抽象词为视觉元素（"温暖"→"暖琥珀色调，柔和漫射光"） |
| 差异化识别度不足 | 回 Q0 重推主体物，检查黑名单 |
| 叙事完整性不足 | 回 Q4 补充人物+处境+转变 |

修正后在评分卡后输出修正版 prompt。最终交付的必须是通过自检的版本。

评分卡格式：
```
【Prompt 评分卡 — 应用商店封面图】
1. 主体清晰度：X/5
...
10. 叙事完整性：X/5
总分：XX/50 [✅ 通过 / ⚠️ 已自动修正]
```

## 生图执行

prompt 自检通过后，先检查环境变量是否已配置 `IMAGE_API` 和 `IMAGE_API_KEY`。

### 未配置

```
未配置生图后端，需要以下信息：

  IMAGE_API=openai         # 后端类型
  IMAGE_BASE_URL=https://api.openai.com   # API 地址
  IMAGE_API_KEY=sk-xxx     # API Key
  IMAGE_MODEL=dall-e-3     # 模型名

请提供，我帮你写入 .env。
```

### 已配置

告知用户当前配置，确认后再生图：

```
当前生图配置：
  IMAGE_API=openai
  IMAGE_MODEL=dall-e-3

确认用此配置生图？（yes/no）
```

用户确认后：

```bash
python3 {baseDir}/scripts/generate.py \
  --prompt "生成的完整中文prompt" \
  --output "output/{应用名}_{时间戳}.jpg"
```

成功后发送图片文件给用户，失败则报告错误。
