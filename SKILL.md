# App 营销绘图 Prompt 生成 & 生图

将 App 营销信息转化为高质量中文绘图 prompt，并完成生图。最终产出图片文件。

## 风格/构图分配机制

### 风格分配（按优先级）
1. **强制指定** — `--style N` 参数直接覆盖
2. **分类推荐** — 分类命中 `CATEGORY_RECOMMENDED_STYLES` 时，按 `行号 mod 风格池大小` 轮选
   - 母婴.儿童：`[4羊毛毡(主推), 9粘土C4D, 3CG厚涂, 2 3D立体]`
   - 其他 25+ 分类各有专属风格池（见 references/category-visual.md）
3. **全局轮换兜底** — 分类未登记时，按 `1→2→3→5→6→7→8→9→10→11→12→1` 顺序（跳过 4；游戏类额外跳过 7/8/12）

**索引位置**：风格池定义在脚本顶部 `CATEGORY_RECOMMENDED_STYLES`；行号即 CSV 中应用所在行的 1-based 行号。

> ⚠️ **游戏类全局禁用风格 7、8、12**（`PARENT_ID == "游戏"` 的全部 11 个子分类：动作冒险 / 飞行射击 / 角色扮演 / 经营策略 / 休闲益智 / 棋牌游戏 / 体育竞速 / 其他游戏 / 特色分类 / 音乐舞蹈 / 打飞机）。无论走哪条路径（强制 / 推荐 / 兜底）都会自动顺延，确保游戏不会渲染玻璃拟态、充气 3D 或 Memphis 卡通风。

### 构图分配
- 风格 12 → 5 种专属 Memphis 居中构图（随机）
- 其他风格 → 优先用 `CATEGORY_RECOMMENDED_COMPOSITIONS` 按行号轮选；未登记则从 10 种通用构图随机取

### 手动覆盖语法
用户附加 `风格=CG厚涂 色调=暖棕色调 构图=宏大仰视式 光线=丁达尔效应 主体=具体描述 品牌层级=高端` 等指令时，跳过对应维度的推导直接使用指定值。批量生成使用 `--style N` 参数强制指定风格。

## 输入信息与 Logo 处理

按优先级提取：应用名称 → 一句话描述 → 应用简介 → 应用类型 → 宣传素材 → 参考图（可选）→ 必须包含元素（可选）→ Logo 图片（可选）

当用户提供 Logo 图片时，从中提取「Logo 视觉指纹」，输出格式：
```
Logo 视觉指纹：
- 主色调：[颜色1]、[颜色2]
- 设计风格：[扁平/3D/线描/...]
- 核心元素：[图标+文字/纯图标/...]
- 品牌气质：[活泼/专业/可爱/...]
```
Logo 提取的主色调和设计风格直接用于 prompt，优先级最高。与分类规则冲突时以 Logo 为准。

## 风格矩阵注入规则

所有 prompt 以 `宽屏16:9横版应用商店封面图` 开头。生成前按以下 5 步执行：

1. **检查手动覆盖（最高优先级）** — 用户附加风格/色调/构图/光线指令，直接使用
2. **分类驱动风格** — 按 `CATEGORY_RECOMMENDED_STYLES` + 行号取模选风格
3. **分类驱动构图** — 按 `CATEGORY_RECOMMENDED_COMPOSITIONS` + 行号取模选构图；未命中分类则从 10 种通用构图随机
4. **品牌色注入** — 优先 ICON 提取（提升为高饱和高明度版本）；ICON 提取失败回退到分类推荐主色
5. **品牌层级修正** — 命中 `PREMIUM_BRANDS` → 高端层级；命中 `QUALITY_BRANDS` → 精品层级；其他 → 标准。母婴.儿童类一律按标准处理

## Prompt 构建规范

### 语言约束

- 全部中文，不得出现英文关键词或括号权重语法
- 不输出负面提示词
- 不得出现具体应用 Logo、品牌标志、商标图案

### 内容安全约束（硬性）

**人物禁令：**
- 默认不出现任何人物，画面主体为物件、符号或场景
- 完全禁止：真人、现实人物肖像、摄影级写实人物、抽象几何剪影、概念艺术插画人物
- 仅当叙事无法用物件/符号表达（如社交、直播类 App 的核心体验就是人与人的连接）时才允许出现人物
- **风格7（玻璃拟态）/ 风格8（充气3D）绝对禁止人物**：不得出现任何人物/人形/人体/人脸/手/肢体，主视觉必须为产品/道具/物件/场景元素
- **风格12（Corporate Memphis）人物规范**：允许人物，必须使用Corporate Memphis风格——小头长肢巨型末端，点状眼极简弧线嘴，肤色米色/浅棕/橙棕（严禁绿/蓝/紫/灰），1.5-2.5pt黑色轮廓线，必须与主体物发生物理接触/重叠并有明确互动动作，约占内容区20%；无法与主体自然关联则省略人物

**法律合规：**
- 禁止具体国家货币符号、币种文字、纸币硬币样式；涉及"货币"主体时必须为中性通用表达
- 禁止知名 IP 形象、版权角色、卡通人物（如米老鼠、皮卡丘等）
- 禁止误导性医疗表述、金融收益承诺、赌博暗示
- 禁止暴力、血腥、色情、歧视性内容
- 禁止 LGBT 相关视觉元素、彩虹旗等符号、女扮男装、男扮女装

**视觉成熟度（硬性）：**
- 除母婴、幼儿类产品外，画面整体气质必须成熟，不得出现低龄化视觉特征
- 低龄化标志（非母婴/幼儿产品禁止）：Q版造型、卡通风格、泡泡字体感、幼儿园配色（高饱和彩虹色乱炖）、幼稚化图形语言

**画面纯净：**
- 禁止中文、英文、数字、Logo、水印、标题文字
- 禁止手机、平板、电脑、电视等电子设备外观（工具类 App 可用屏幕内容，但不能出现完整设备外形）
- 禁止 AI 味的图片特征：过度光滑塑料感皮肤、诡异打光、千篇一律的AI艺术脸、过度对称构图、不自然的完美感
- 禁止画中画、分屏、多画幅拼贴、内嵌相框

### 结构模板

`[主体+动作]，[构图/取景]，[场景/环境]，[风格]，[光线]，[技术参数]`

光线模块对生图影响极大，每个 prompt 必须包含具体光线描述。

### 正向质量词

每个 prompt 必须注入（从根源上规避低质量画面）：

**画质保证词（必选）：** 高清，画面清晰锐利，构图完整，主体完整呈现，精准对焦

**质感提升词（按场景选择）：**
- 防止低龄幼稚：成熟视觉风格、专业插画质感
- 防止廉价感：精致质感、高端视觉、细节丰富
- 防止负面氛围：正向情绪、活力氛围、积极场景

**负面约束词（应用商店封面图必选）：** 禁止出现任何文字、禁止出现标题、禁止出现副标题、禁止出现宣传语

### 参考图与元素约束

有参考图时，prompt 末尾追加：`参考附图的整体风格、色调与构图氛围`
有必须包含元素时追加：`画面中必须包含：[用户输入的元素描述]`

### 画面质量三原则

1. **防止画面零碎** — 前景、中景、背景各只一个主要元素，必须包含留白区域
2. **场景构图策略** — 应用商店封面图：宏大场景感，主体占比多档可选（15-20%/45-60%/75-90%）
3. **突出主体** — 主体有明确打光（边缘光/逆光/聚光），背景虚化，形成色彩或明暗对比

人物禁现原则（风格1-6, 9-10）：默认画面只有物件/符号/场景，不出现任何人物；如确需人物只展示背影/侧身/局部，严禁正脸。
人物完全禁止（风格7/8）：不得出现任何人物/人形/人体/人脸/手/肢体。
人物开放（风格11）：扁平插画风格，无正脸限制。
人物开放（风格12）：Corporate Memphis风格，小头长肢巨型末端，点状眼极简弧线嘴，肤色米色/浅棕/橙棕，必须与主体物物理接触/重叠并有明确互动动作；无法与主体自然关联则省略。

## 品牌层级识别

在分类视觉指纹之上叠加品牌层级修正（优先级仅次于 Logo 提取）：

| 层级 | 识别信号（在 `应用名+简介+描述` 中匹配） | 修正方向 |
|------|----------------------------------|---------|
| 高端 | 大厂：腾讯/网易/字节/米哈游/阿里/支付宝/百度/华为/小米/京东/美团/拼多多/抖音/快手/B站；央媒/国企：央视/新华社/人民日报/中国移动联通电信/工商建设农业招商银行/国家(反诈)；硬件厂：OPPO/vivo/联想 | 注入：电影级质感、精致光影、高端商业视觉、克制配色、专业摄影感构图。禁用：卡通、Q版、糖果色、可爱、萌、幼稚 |
| 精品 | 国际授权：迪士尼/Disney/皮克斯/Pixar/BBC/国家地理/牛津/剑桥/DK/RAZ；优质教育：新东方/好未来/学而思/宝宝巴士/凯叔讲故事/洪恩 | 注入：精品插画感、国际化视觉风格、成熟视觉调性、专业品质感。禁用：卡通、Q版、可爱、萌、幼稚、糖果色 |
| 标准 | 无以上信号；**母婴.儿童类一律标准**（避免与卡通基调冲突） | 不做额外修正 |

识别成功时在 prompt 末尾注入对应修正词并强制排除禁用词。用户可手动覆盖：`品牌层级=高端/精品/标准`。

层级与分类的优先级：Logo提取 → 品牌层级修正 → 分类驱动风格/构图 → 风格池（兜底）

## 信息补全机制

核心原则：不追问用户，全部自主推断，推断过程透明化。

### Step 1：信息分级

| 等级 | 判断标准 | 处理策略 |
|------|---------|---------|
| 充足 | 名称+分类+简介 | 直接生成，标注推断项 |
| 一般 | 名称+分类 | 名称语义解析+分类惯例，补全后生成 |
| 稀少 | 只有名称或只有分类 | 双引擎推断，补全后生成 |
| 极少 | 一句模糊描述 | 生成3个差异化风格方向供选择 |

### Step 2：名称语义解析

| 名称关键词 | 推断方向 | 视觉倾向 |
|---------|---------|---------|
| 龙/兽/精灵/王国/魔法/神/战 | 奇幻冒险游戏 | 高饱和暖色、史诗插画、戏剧性构图 |
| 学/教/宝/启/课/题/考 | 教育/儿童 | 柔黄嫩绿、卡通插画、亲切温暖 |
| 钱/财/金/贷/理/股/账 | 金融商务 | 深蓝金色、简洁专业、信任感 |
| 聊/友/圈/社/群/话/见 | 社交 | 珊瑚橙暖粉、人物互动、温暖活跃 |
| 音/乐/歌/听/播/声/曲 | 音乐娱乐 | 律动紫蓝、艺术感、动感视觉 |
| 英文/字母缩写/数字组合 | 科技工具 | 冷色系、极简UI感、科技感 |

名称无法匹配时，根据整体语感推断（硬朗→游戏/工具；柔软→生活/社交；严肃→商务/金融）。

### Step 3：极简兜底

信息极少时，生成3个有明显差异的风格方向（色调/风格/氛围各不相同），供用户选择。

### Step 4：推断透明化

每次生成前输出「信息解析卡」：
```
信息解析卡：
- 已知：[用户提供的信息]
- 推断：[每项推断内容及依据]
  · 游戏类型 → 开放世界RPG（依据：名称含"王国"）
  · 品牌主色 → 深紫+金色（依据：游戏-开放世界分类推荐色）
  · 视觉风格 → 史诗概念艺术（依据：一级分类"游戏"渲染风格）
  · 品牌层级 → 标准层级（依据：无触发词）
```

优先级：Logo提取 > 用户明确说明 > 三级分类推荐色 > 名称语义推断 > 一级分类基础

### Step 5：主体物语义推导

**严格按以下优先级执行，不得跳级：**

**① 优先 — S_INTRO 提取2个具体可视化词**
从应用简介（S_INTRO）中提取2个具体可视化名词（2-6字中文词组），过滤 VISUAL_STOPWORDS 中的抽象/功能/品牌词，组合为主体描述：
```
S_INTRO提取词1 + 与 + S_INTRO提取词2 [+ HUMAN_DESC≤20字补充场景氛围]
```

**② S_INTRO 只有1个有效词时**
结合 HUMAN_DESC（≤20字，无机构标签）补充场景：
```
S_INTRO提取词 + HUMAN_DESC场景描述
```

**③ S_INTRO 无有效词时 — 根据 NAME 分类提取 HUMAN_DESC**
使用 HUMAN_DESC（≤24字，无机构标签）+ App名产品场景：
```
HUMAN_DESC + App名产品场景
```

**④ SUBJECT_POOL 分类主体池兜底**
按 NAME 分类从 references/category-visual.md 的 SUBJECT_POOL 随机取一条

**⑤ 通用兜底**
```
{App名}核心功能视觉符号
```

---

**过滤规则（所有层级通用）：**
- VISUAL_STOPWORDS：过滤"软件/体验/平台/功能/工具/系统/管理/服务/用户/安卓/官方/腾讯/百度"等80+词
- LABEL_PATTERNS：HUMAN_DESC 含"官方/安卓/手机银行/国家税务/旗下/APP"时整句丢弃
- GENERIC_TAGS：S_INTRO 提取时跳过"常用工具/其他工具/休闲娱乐/数字人"等通用分类词

---

**Q0-Q4 差异化深度推导**（适用于单张精品图或信息充足时）：

**Q0. 差异化过滤** — 这个 App 和同类相比最独特的功能/体验是什么？

行业通用符号可用但需优化精致度（场景层次、光影、材质），避免 icon 感：

| 分类 | 通用符号 | 优化方向 |
|------|---------|---------|
| 房产 | 钥匙、房子轮廓、握手 | 加入场景/质感/光影提升精致度 |
| 教育 | 书本、铅笔、灯泡 | 加入人物/情感/叙事 |
| 金融 | 金币、钱袋、上涨箭头 | 加入场景/情绪/叙事感 |
| 音乐 | 耳机、音符、麦克风 | 加入人物/沉浸感 |
| 社交 | 对话气泡、握手、人群 | 加入人物/真实社交场景 |

**Q1. 核心动作** — 用户使用时在做什么？
**Q2. 代表物件** — 最具代表性的视觉元素（必须通过 Q0 过滤）
**Q3. 视觉化方向** — 如何表达最能传达价值和情绪？
**Q4. 叙事三要素** — 把功能转化为可读故事：
- 人物 → 目标用户具体形象（年龄/状态/处境）
- 处境 → 使用 App 前后的关键节点
- 转变 → 画面定格的那一秒（问题被解决的瞬间或目标达成的状态）

**视觉化变体生成**（每个变体必须包含至少3层场景、2个动作、3层光影、2个材质、2个氛围词）：

| 变体 | 叙事节点 | 画面定格 |
|------|---------|---------|
| A | 痛点时刻 | 使用 App 前，问题最尖锐的那一秒 |
| B | 转变时刻 | 使用 App 中，问题被解决的瞬间 |
| C | 结果时刻 | 使用 App 后，目标达成的情感状态 |

输出格式：
```
· S_INTRO提取词 → [词1] + [词2]（或兜底层级说明）
· 差异化功能 → [Q0推导结果]
· 核心动作 → [Q1推导结果]
· 代表物件 → [Q2推导结果]（主体物）
· 叙事三要素 →
  · 人物：[目标用户具体形象]
  · 处境：[关键节点描述]
  · 转变：[画面定格的那一秒]
· 视觉化变体 →
  · 变体A：[痛点时刻]
  · 变体B：[转变时刻]
  · 变体C：[结果时刻]
· 人物角色 → [保留/省略] + 理由
· 本次使用 → 变体[X]（由主体物索引 mod 3 决定）
```

## 质量自检（人工/可选）

脚本当前不自动执行质量自检。若需评审，按以下 10 维度对生成结果打分（每项 0-5 分，总分 50）：

1. 主体清晰度（是否有具体可画视觉主体）
2. 光线描述（是否有具体光线词，如"戏剧性魔法逆光"）
3. 构图完整性（是否符合分类推荐构图）
4. 风格一致性（风格词是否与分类匹配）
5. 构图场景感（图标用宏大感、海报用极简居中等）
6. 品牌色落地（是否用了 ICON 提取的具体色名）
7. 可生成性（每个词是否有视觉对应物）
8. Logo 禁用（是否出现 Logo/App 图标/界面）
9. 差异化识别度（是否能一眼识别 App 核心功能）
10. 叙事完整性（人物+处境+转变三要素是否清晰）

总分 <35 → 修改 prompt 后重跑。常见修正动作：
- 主体抽象 → 替换为具体可视化场景
- 缺光线 → 追加"戏剧性逆光"
- 风格不一致 → 改用 `CATEGORY_RECOMMENDED_STYLES` 中对应风格的描述词
- 通用符号无场景感 → 检查 SUBJECT_POOL 黑名单后重推

## 生图执行

prompt 自检通过后，先检查环境变量是否已配置 `IMAGE_API` 和 `IMAGE_API_KEY`。

### 未配置

```
未配置生图后端，需要以下信息：

  IMAGE_API=openai         # 后端类型
  IMAGE_BASE_URL=https://api.openai.com   # API 地址
  IMAGE_API_KEY=sk-xxx     # API Key
  IMAGE_MODEL=dall-e-3     # 模型名

请提供，我帮你写入 .env。
```

### 已配置

告知用户当前配置，确认后再生图：

```
当前生图配置：
  IMAGE_API=openai
  IMAGE_MODEL=dall-e-3

确认用此配置生图？（yes/no）
```

用户确认后：

```bash
python3 {baseDir}/scripts/generate.py \
  --prompt "生成的完整中文prompt" \
  --output "output/{应用名}_{时间戳}.jpg"
```

成功后发送图片文件给用户，失败则报告错误。
